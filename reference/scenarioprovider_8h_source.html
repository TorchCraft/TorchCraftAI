127 /**
128  * Generates Random armies. Parameters:
129  * AllowedRaces: the set of races we can randomly draw from
130  * maxSupplyMap: Maximum supply for each race.
131  * randomSize: if true, the target supply is drawn uniformly in [min(10,
132  * maxSupply), maxSupply]. Else, the target supply is the one
133  * from the supplyMap.
134  * checkCompatiblity: if true, we don't sample armies that are incompatible.
135  * Sources of incompatiblilites: air units in one army but no air weapon in
136  * the other, ground units in one army but no ground weapon in the other,
137  * cloaked/burrowable units in one army but no detector in the other.
138  * In other words, we require that each unit can be attacked by at least one
139  * unit of the other army.
140  *
141  * Note that due to sampling artifacts, the actual sampled supply
142  * might be a bit smaller than the target
143  *
144  * The default parameters give scenarios that are somewhat balanced (as
145  * mesured by playing random battles using attack-closest heuristic and no
146  * micro). Protoss has a slightly lower win-rate on average, around 30%.
147  *
148  * The following units are currently left out:
149  * - All spell caster (except Science Vessels, which are used as detectors for
150  * terrans)
151  * - Reavers (no way to spawn scarabs currently)
152  * - Carrier (same with interceptors)
153  * - Dropships
154  * - SCVs, Drones, Probes
155  * - Scourge + infested terrans (annoying micro)
156  */
